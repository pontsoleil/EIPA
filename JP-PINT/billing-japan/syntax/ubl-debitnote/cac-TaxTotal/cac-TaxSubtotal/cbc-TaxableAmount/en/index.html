4.4 Cardinality assesment

CAR-1
Semantic: optional(0..x) → Syntax: mandatory(1..x)
[Issue] If the value is not present, the UBL schema validation reports an error.
[Resolution] Agree on "default value if missing" (e.g. 0, 1-1-1970, AAA)'
CAR-2
Semantic: mandatory(1..x) → Syntax: optional(0..x)
[Issue] None.
[Resolution] Add a rule in the schematron that the element shall be present.
CAR-3
Semantic: single(x..1) → Syntax: multiple(x..n)
[Issue] None.
[Resolution] Add a rule in the schematron that the element shall not be repeated.
CAR-4
Semantic: multiple(x..n) → Syntax: single(x..1)
[Issue] Repeating elements cannot be handled.
[Resolution]
  1) If possible, repeat a higher level in the structure.
  2) In the case of text elements, concatenate the repeating elements.
CAR-5
Semantic: element missing → Syntax: element mandatory
[Issue] Yes.
[Resolution] Agree on "default value if missing" (e.g. 0, 1-1-1970, AAA)

    cbc:TaxableAmount

    Sum of all taxable amounts subject to a specific TAX category code and TAX category rate (if the TAX category rate is applicable).

    Namespace
    cbc urn:oasis:names:specification:ubl:schema:xsd:CommonBasicComponents-2
    XPath
    /Invoice/cac:TaxTotal/cac:TaxSubtotal/cbc:TaxableAmount
    Data type
    AmountType
    Cardinality
    1..1  
    Business Term ID
    ibt-116
    Business Term
    TAX category taxable amount
    Additional Explanation
    In the numerical representation, for data type amount, only the integer is allowed and the integer part is not separated by 1000 units.Tax category code is for identifying the attribute of tax category in Japan Consumption Tax scheme, like standard rate, reduced rate, etc. The operation in Japan is as follows.
    S Standard rate Consumption Tax
    AA Lower rate Consumption Tax
    E Exempt from Tax
    G Free export item, Consumption Tax not charged
    O Outside scope of tax (The seller is not registered for Consumption Tax.)
    Example
    87700
