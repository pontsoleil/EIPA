    Japanese Summary Invoice, pattern 1, message specification

    NOTE: All element names are inhereted from Peppol International Invoicing (PINT) and naming use the term invoice, the same items are used in standard commercial invoice, summarised invoice and delivery note (debit note). The difference is Business process type (Profile ID) and Specification identifier (Customization ID). The tag names are correct according to the UBL 2.1 Invoice schema.

      XML Datatype Card Business Term / Description
    Invoice InvoiceType 1..n

    Invoice
      • cbc:CustomizationID IdentifierType 0..1

    Specification identifier (Customization ID)
    An identification of the specification containing the total set of rules regarding semantic content, cardinalities and business rules to which the data contained in the instance document conforms.

    Default value: urn:peppol:pint:billing-3.0@jp:peppol-1@jp:suminvpt1-1
      • cbc:ProfileID IdentifierType 0..1

    Business process type (Profile ID)
    Identifies the business process context in which the transaction appears, to enable the Buyer to process the Invoice in an appropriate way.

    Default value: urn:peppol:bis:billing
      • cbc:IssueDate DateType 1..1

    Invoice issue date
    The date when the Invoice was issued.

    Example value: 2003-03-04
      • cbc:IssueTime TimeType 1..1

    Invoice issue time
    The time of day when an invoice was issued

      • cbc:DueDate DateType 0..1

    Payment due date
    The date when the payment is due.

    Example value: 2003-03-20
      • cbc:InvoiceTypeCode CodeType 0..1

    Invoice type code
    A code specifying the functional type of the Invoice.

    Example value: 380 (Commercial Invoice)
      • cbc:Note TextType 0..n

    Invoice note subject code
    The subject of the textual note in EN BT-22.

      • cbc:Note TextType 0..n

    Invoice note
    A textual note that gives unstructured information that is relevant to the Invoice as a whole.

      • cbc:TaxPointDate DateType 0..1

    TAX point date
    The date when the TAX becomes accountable for the Seller and for the Buyer in so far as that date can be determined and differs from the date of issue of the invoice, according to the TAX directive.

      • cbc:DocumentCurrencyCode CodeType 0..1

    Invoice currency code
    The currency in which all Invoice amounts are given, except for the Total TAX amount in accounting currency.

    Example value: JPY
      • cbc:TaxCurrencyCode CodeType 0..1

    Tax accounting currency
    The currency used for TAX accounting and reporting purposes as accepted or required in the country of the Seller.

      • cbc:AccountingCost TextType 0..1

    Buyer accounting reference
    A textual value that specifies where to book the relevant data into the Buyer's financial accounts.

      • cbc:BuyerReference TextType 0..1

    Buyer reference
    An identifier assigned by the Buyer used for internal routing purposes.
