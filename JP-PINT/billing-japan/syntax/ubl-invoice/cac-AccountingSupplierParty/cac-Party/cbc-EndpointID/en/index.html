4.4 Cardinality assesment

CAR-1
Semantic: optional(0..x) → Syntax: mandatory(1..x)
[Issue] If the value is not present, the UBL schema validation reports an error.
[Resolution] Agree on "default value if missing" (e.g. 0, 1-1-1970, AAA)'
CAR-2
Semantic: mandatory(1..x) → Syntax: optional(0..x)
[Issue] None.
[Resolution] Add a rule in the schematron that the element shall be present.
CAR-3
Semantic: single(x..1) → Syntax: multiple(x..n)
[Issue] None.
[Resolution] Add a rule in the schematron that the element shall not be repeated.
CAR-4
Semantic: multiple(x..n) → Syntax: single(x..1)
[Issue] Repeating elements cannot be handled.
[Resolution]
  1) If possible, repeat a higher level in the structure.
  2) In the case of text elements, concatenate the repeating elements.
CAR-5
Semantic: element missing → Syntax: element mandatory
[Issue] Yes.
[Resolution] Agree on "default value if missing" (e.g. 0, 1-1-1970, AAA)

    cbc:EndpointID

    Identifies the Seller's electronic address to which the application level response to the invoice may be delivered.

    Namespace
    cbc urn:oasis:names:specification:ubl:schema:xsd:CommonBasicComponents-2
    XPath
    /Invoice/cac:AccountingSupplierParty/cac:Party/cbc:EndpointID
    Data type
    IdentifierType
    Cardinality
    0..1  
    Business Term ID
    ibt-034
    Business Term
    Seller electronic address
    Additional Explanation
    In case of Japan, it is supposed that ibt-029, ibt-030, ibt-031 and ibt-034 are used in the following way. ibt-029 is used to set identifier issued by numbering agencies (such as JIPDEC, TDB, TSR), ibt-030 is used to set identifier issued by government/tax authority, ibt-031 is used to set Japan consumption tax registration number, and ibt-034 is used to set user ID of Peppol eDelivery Network.
    Example
    JP1234567890123
